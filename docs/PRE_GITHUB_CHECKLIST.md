# 上传 GitHub 前检查清单

## A. 内容筛选

- [ ] 确认是否包含敏感信息（账号、路径、隐私数据、许可证受限数据）。
- [ ] 确认是否需要公开大型数据文件（CSV/Parquet/MP4/PDF/ZIP）。
- [ ] 对不需要公开的大文件，删除或迁移到外部存储。

## B. 仓库可读性

- [ ] 根目录 `README.md` 信息完整（项目目标、结构、运行方式、注意事项）。
- [ ] 子项目至少有一个可运行入口说明（脚本或 Notebook）。
- [ ] 图片和结果文件命名有含义，避免 `test_final2` 这类命名。

## C. 代码与路径

- [ ] 路径尽量使用相对路径或 `pathlib.Path`。
- [ ] Notebook 中删除无关调试输出（大表格、超长日志）。
- [ ] 确认脚本在仓库根目录下可定位输入输出。

## D. Git 规范

- [ ] `.gitignore` 生效（缓存、临时目录、系统文件未被追踪）。
- [ ] 如需保留大文件，先配置 Git LFS。
- [ ] 首次提交说明清晰（例如：`init: research repository structure and docs`）。

## E. 发布后体验

- [ ] 打开仓库首页时，README 前 30 秒可说明“做什么、怎么跑、看哪里”。
- [ ] 关键目录点击后不迷路（有 README 或文档指引）。
